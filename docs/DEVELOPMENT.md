# Development Guide

## Requirements

- Windows 10 or Windows 11.
- .NET 8 SDK.
- Visual Studio 2022 or a compatible editor.
- Optional: WinGet for software maintenance workflows.
- Optional: Node.js for regenerating brand assets.

## Clone

```powershell
git clone git@github.com:jaysonguglietta/spacepilot.git
cd spacepilot
```

## Build

From the repository root:

```powershell
dotnet restore .\SpacePilot.sln
dotnet build .\SpacePilot.sln
dotnet run --project .\src\SpacePilot\SpacePilot.csproj
```

The app targets WPF, so full build and runtime validation should happen on Windows.

## Publish Locally

Framework-dependent publish:

```powershell
dotnet publish .\src\SpacePilot\SpacePilot.csproj -c Release -r win-x64 --self-contained false -o .\artifacts\publish\SpacePilot
```

Self-contained publish:

```powershell
dotnet publish .\src\SpacePilot\SpacePilot.csproj -c Release -r win-x64 --self-contained true -o .\artifacts\publish\SpacePilot-self-contained
```

## Assets

The editable logo source is:

```text
src\SpacePilot\Assets\AppLogoMark.svg
```

The Windows `.ico` file is generated by:

```powershell
node .\scripts\generate-brand-assets.mjs
```

The generator writes:

```text
src\SpacePilot\Assets\AppIcon.ico
```

## Coding Notes

- Keep cleanup rules explicit and reviewable.
- Route file-changing behavior through safety checks, protected-path checks, quarantine, and receipts.
- Do not add automatic registry cleaning.
- Do not add silent uninstall, startup-disable, or browser-state cleanup behavior.
- Keep Windows-specific calls isolated in services.
- Prefer clear UI text over hidden automation.

## Validation

Before a release candidate:

```powershell
dotnet build .\SpacePilot.sln -c Release
dotnet test .\tests\SpacePilot.Tests\SpacePilot.Tests.csproj -c Release
```

Automated tests cover path safety, cleanup rules, quarantine, receipts, preferences, and WinGet parsing. Before release, also manually validate:

- First-run safety note.
- Scan and cleanup with quarantine enabled.
- Restore from quarantine.
- Purge quarantine.
- Receipt export.
- Large-file scan.
- Duplicate-file scan.
- Browser profile discovery.
- WinGet update discovery.
- Startup inventory.
- Settings audit.
- Weekly scan reminder enable/disable.

## CI And Packaging

GitHub Actions runs restore, build, tests, publish, and release zip packaging on Windows. See [Continuous Integration](CI.md).

Create a local package:

```powershell
.\scripts\package-spacepilot.ps1 -Configuration Release -Runtime win-x64 -SkipSigning
```

Create an MSI when WiX is installed:

```powershell
.\scripts\build-msi.ps1 -SkipSigning
```

## Recommended Test Fixtures

Use disposable folders and test profiles when validating cleanup behavior. Do not test destructive flows against important personal data.
